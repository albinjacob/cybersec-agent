3. DECISIONS MADE
- [Detection is deterministic; LLM only narrates] → Rule-based/regex/static-analysis findings are extracted BEFORE any LLM call in every agent. The LLM's only job is turning structured findings into readable prose.
- [Findings rendered as color-coded cards, not one text blob] → Cards are built directly from each agent's structured Python data, not from LLM text.
- [Every finding card gets a deterministic "💡 What this means" + "✅ Recommended Action"] → From a hardcoded `KNOWLEDGE_BASE` dict, NOT LLM-generated, so guidance is identical regardless of model. Real CVEs get an NVD "Learn More" link; unknown non-CVE ids get neutral "best-practice violation" framing.
- [Real LangGraph over hand-rolled SimpleGraph] → `orchestrator.py` uses `langgraph.graph.StateGraph` with genuine parallel fan-out: Log Monitor + Vulnerability Scanner both edge from START, join at Threat Intelligence, then sequential Incident Response → Policy Checker.
- [BYOK for both LLM reasoning and embeddings, each with its own settings section] → `agents/llm.py`'s `configure(provider, api_key, model)` and the new `agents/embeddings.py`'s `configure(provider, api_key, model)` are architecturally identical patterns but deliberately kept as SEPARATE UI sections ("⚙️ Model Settings" vs "🔎 Policy RAG Embeddings") in the same right sidebar, since they control different things - LLM settings affect narrative reasoning across all 5 agents, embedding settings affect only Policy Checker's RAG retrieval.
- [OpenRouter default model = openai/gpt-4o-mini] → Don't make users blindly pick the top of a 344-model list.
- [Multi-page nav UI] → Left `gr.Sidebar` navigates Overview (5 agent cards + blurbs) + one page per agent + Full Report, each agent with its OWN Quick Demo/Analyze Your Own Files.
- [Every agent page's Run button still executes the FULL pipeline] → No agent can run in true isolation (data dependencies). One shared `ALL_OUTPUTS` list means any page's run populates every other page too.
- [Reasoning badge + severity counts + cards merged into ONE HTML component per page] → Avoids redundant/overlapping "processing" loading indicators.
- [Custom `.subdued-text` CSS class instead of Gradio's theme variable] → Gradio's own subdued-text variable is non-adaptive/broken between light/dark.
- [Real Trivy + real NVD, both with automatic fallback] → `scan_mode` ("trivy"/"static-fallback") and `feed_mode` ("live-nvd"/"local-fallback"), mirroring `reasoning_mode`.
- [NVD keyword order: dependency/package names first, log-derived phrases last] → Precise names surface far more relevant CVEs than broad phrases within the `NVD_MAX_KEYWORDS=5` cap.
- [Accepted, not fixed: naive NVD keyword collisions] → e.g. "flask" also matches unrelated Xen "FLASK" CVEs. Proper fix needs CPE-based matching - deliberately out of scope.
- [No vector database for the RAG upgrade] → NIST 800-53 corpus (~1,023 chunks incl. the ISO/SOC2 excerpt, ~2MB of cached vectors) is orders of magnitude below the scale where Pinecone/Weaviate/Qdrant-style ANN indexes matter. Plain in-memory `sklearn.metrics.pairwise.cosine_similarity` is sufficient - confirmed working in production.
- [Embeddings precomputed/cached, rebuilt via a manual UI button, not recomputed every run] → NIST revises 800-53 roughly once every several years; re-embedding on every pipeline run would be pure waste.
- [Embedding cache validated against corpus hash AND embedding model/provider before trusting it] → Different models produce incompatible vector spaces. A mismatch shows a warning in the UI and the loader falls back to TF-IDF rather than silently using incompatible vectors.
- [`sentence-transformers` (local, `all-MiniLM-L6-v2`) as the always-available default; OpenAI as a BYOK alternative] → Local needs zero configuration/cost, matching the project's zero-setup philosophy; unlike `agents/llm.py`, there's no "mock" tier for embeddings since local already IS the zero-cost always-available option.
- [`EMBEDDING_MIN_SCORE = 0.25`, empirically calibrated, not guessed] → Tested real cosine similarity scores: unrelated text tops out ~0.17, genuine (if imperfectly worded) matches start ~0.24, strong matches run 0.4-0.55+. 0.25 sits safely above the noise floor.
- [UI overhaul: streaming execution + dashboard + theme, Gradio-native, no React] → User asked for "modern, intuitive, professional, architecturally sound." Implemented as 4 packages: (A) streaming pipeline + live DAG tracker, (B) custom theme/fonts/top bar, (C) dashboard Overview, (D) findings UX. Deliberately stayed in Gradio/Python - no framework migration - consistent with the project's original deliberate Gradio choice.
